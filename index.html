Lab Challenges

Problem 1 - Cybershujaa

Problem Statement: Perform OSINT reconnaissance on a target domain to enumerate public-facing assets, subdomains, employee emails, and exposed code repositories without violating scope.

Approach: Use passive information-gathering techniques: run subdomain enumeration, search certificate transparency logs, query public WHOIS/DNS, search social networks and Git hosting for leaked secrets, and produce an asset inventory.

Tools Used: whois, dig, crt.sh, subfinder/amass, theHarvester, Google dorking, LinkedIn/X searches, GitHub search.

Key Lessons Learned: Passive OSINT yields many exposed assets; certificate transparency and Git search are high-value.

Problem 2 - Cybershujaa

Problem Statement: Scan and enumerate a lab network to identify live hosts, open services and versions, then prioritize likely vulnerable targets.

Approach: Perform ICMP/TCP discovery to find live hosts, run port scans, detect versions, and correlate results with known CVEs.

Tools Used: nmap, masscan, nmap scripts, Vulners lookup.

Key Lessons Learned: Version accuracy speeds exploitation planning; automate CVE lookups but manually verify exploitability.

Problem 3 - Cybershujaa

Problem Statement: Identify and exploit a SQL injection vulnerability in a lab web application, extract limited data, and prepare a remediation report.

Approach: Manual input testing, intercept requests with Burp, confirm injection, extract safe data, and recommend parameterized queries.

Tools Used: Burp Suite, sqlmap, browser DevTools.

Key Lessons Learned: Manual testing gives context automation misses.
